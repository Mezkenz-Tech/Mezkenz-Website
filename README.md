# Atlas IT Solutions Website

A production-ready marketing site for Atlas IT Solutions built with Next.js (App Router) and Tailwind CSS.

## Getting Started

```bash
npm install
npm run dev
```

The app will be available at [http://localhost:3000](http://localhost:3000).

### Environment variables

- `GA_MEASUREMENT_ID` (optional): when set, Google Analytics is injected automatically.

## Project Structure

- `app/` – Next.js App Router pages and API routes.
- `app/_data/` – JSON content for services, products, pricing, FAQs, testimonials, and blog posts.
- `components/` – Reusable UI components including the header, footer, contact form, and product card.
- `sections/` – Page section components for the marketing site.
- `public/` – Static assets such as Open Graph and illustration images.
- `tests/` – Vitest unit tests for core components.

To edit site content, update the JSON files in `app/_data/`. The pages and sections reference these files directly, so edits appear automatically after a reload.

## Testing

```bash
npm run test
```

Vitest + Testing Library are configured for component-level tests.

## Deployment (Firebase Hosting)

1. Install the Firebase CLI if you haven't already:
   ```bash
   npm install -g firebase-tools
   ```
2. Log in and initialize hosting:
   ```bash
   firebase login
   firebase init hosting
   ```
   - Select **Use an existing project** (or create a new one).
   - Set the public directory to `.next`.
   - Choose **No** for single-page app rewrite.
   - Configure builds to run `npm run build` before deploy.
3. Build the project:
   ```bash
   npm run build
   ```
4. Deploy:
   ```bash
   firebase deploy --only hosting
   ```

Firebase will serve the statically optimized Next.js build with dynamic routes handled via SSR functions automatically generated by Firebase.
